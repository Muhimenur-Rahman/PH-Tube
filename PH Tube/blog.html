Var :

  • Scope: Function-scoped, meaning it is accessible within the function it was declared in, or globally if declared outside any function.
  • Hoisting: Variables declared with var are hoisted to the top of their scope and initialized with undefined.

    • Let :

  • Scope: Block-scoped, meaning it is only accessible within the block (e.g., { }) it was declared in.
  • Hoisting: let declarations are also hoisted, but it cannot be used before the declaration line. Otherwise it will result in an error.

    • Const :

  • Scope: const variables are also only available within the block where they are declared, similar to let. They must be assigned a value when declared and cannot be reassigned.
  • Hoisting: Similar to let, const declarations are hoisted but cannot be used before the declaration line. They must also be initialized at the time of declaration.

    • Undefined :

  • When a variable is declared but not assigned a value, it is undefined.
  • If a function is called with fewer arguments than it is defined to accept, the missing parameters are undefined.
  • Accessing a property that does not exist on an object returns undefined.
  • Elements of an array that are not explicitly assigned values are undefined.

    • Null :

  • null is used to explicitly indicate that a variable should have no value.
  • null is used to clear an object reference, indicating that it no longer points to any object.
  • null is used in DOM APIs to represent the intentional absence of a node.

    • Rest API :

    A REST API is a method for various software applications to interact over the internet by adhering to specific guidelines. These guidelines are based on the REST architectural style, which ensures that communication remains simple and efficient. REST APIs enable programs to exchange data using straightforward web addresses, facilitating easy requests and data sharing. For instance, a client application might request data, such as a list of users, from a server by sending a request to a particular URL. The server then replies with the requested data, typically in a format like JSON. This communication method allows different programs to work together seamlessly over the web, making it easy to share data and functionality.
